npm包管理

什么是npm

npm是随同NodeJS一起安装的包管理工具，只要安装了node,我们就可以通过命令行使用npm

启动命令行，运行 npm -v 就可以看到npm的版本号，根据安装的node版本不同，内部集成的npm也不同，只要能显示一个版本号，就表示npm可用。

npm卸载：你卸载node的时候，一起卸载了

npm作用：

  • 允许用户从NPM服务器下载别人编写的第三方模块到本地使用。
  • 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  • 允许用户将自己编写的模块或命令行程序上传到NPM服务器供别人使用。

下载别人的第三方包到本地

  • 使用 npm install 包名 下载模块

    1:打开命令行，切换到你需要下载第三方包的目录

    2:输入指令 npm install 包名。 简写 npm i 包名

    3：当前目录会多一个 node_modules的文件夹，里面就有jquery的全部文件

    注意：会自动创建node_modules 文件夹

    ​ 默认自动下载的是当前这个包的最新版本

  • 下载指定版本号的第三方包

    打开命令行，切换到你需要下载第三方包的目录，输入指令 npm install 包名@版本号

    简写： npm i 包名@版本号

    会下载指定版本的包

    注意：一份node_modules文件夹里面，一个包只能存在一个版本，当你下载第二次的时候会把前面的自动卸载

npm初始化。

每个项目的根目录下面，会按照你的要求在当前目录下生成一个package.json文件，用来记录你的整个项目，是项目的描述文件，里面记录这个项目所需要的各种模块，以及项目的配置信息（比如名称、版本、许可证等元数据），初始化以后，使用npm安装的每一个包，都会记录

打开命令行，切换到项目目录

步骤：输入指令 npm init；

会进入选择输入列表，按照规则填写就可以了（package name 不能使用中文）

创建package.json

package.json文件的作用

  • 列出您的项目所依赖的软件包

  • 指定项目可以使用的软件包的版本

  • 使您的构建具有可复制性，因此更易于与其他开发人员共享

package.json重要字段

  • dependencies: 生产环境依赖模块 正式环境 线上环境 项目上线后的环境

  • devDependencies ： 开发环境依赖模块 项目上线前开发过程的环境

  • scripts: 脚本

    scripts中存储的都是命令的别名，当我们频繁执行的命令比较长的时候，我们就可以将这个比较长的命令写在这个命令之中，设置一个别名，执行任务时 直接使用 npm run 别名

    • 默认情况下会创建一个空的测试脚本
    • “ scripts”属性是一个字典，其中包含在包的生命周期中的不同时间运行的脚本命令。 关键是生命周期事件，而值是此时要运行的命令

package-lock.json文件的作用

​ 作用：这个文件会详细的记录模块与模块之间的依赖关系，以及模块的版本和模块的下载地址

优点：

  1. 锁定包的版本，确保再次下载时不会因为包版本不同而产生问题
  2. 加快下载速度，因为该文件中已经记录了项目所依赖第三方包的树状结构和包的下载地址，重新安装时只需下载即可，不需要再去一一查看包的依赖关系了，直接下载就可以了

清除缓存

在下载包时怎么下载都下载不下来，因npm是国外的我们其实是去国外转了一圈再回来，下载和网络也有关系，之前下载过这个包，或者之前下载失败了有半个包，每次下载都是半个，需要清除缓存

  • 清除下载记录

    打开命令行，在那个目录无所谓

    输入指令 npm cache clear -f 是清除下载包的记录

  • 强制清除缓存

    清除本地下载记录，比如说在下载的时候没有网了，只下载了一半，以后每一次下载都是这一半

    方式:c:/Users/你自己名字/AppData/Roaming直接删除里面的npm-cache文件夹 即可

模块加载机制

当模块拥有路径且拥有后缀时

通过路径，直接引入模块,如果完整路径，直接引入模块

当模块拥有路径但没有后缀时

省略后缀

  1. 先找该路径下同名JS文件，如果没有再找同名JS文件夹

  2. 如果找到了同名文件夹（find文件夹），找文件夹中的index.js

  3. 如果找到的同名（find）文件夹中没有index.js就会去当前文件夹中的package.json文件中查找main选项中的入口文件

  4. 如果找指定的入口文件不存在或者没有指定入口文件就会报错，模块没有被找到

当模块没有路径且没有后缀时

  1. Node.js会查看它是否为系统内置的系统模块
  2. 如果不是内置的系统模块，会假设它为第三方模块，去node_modules文件夹中查找
  3. 先看是否有同名的js文件
  4. 如果没有同名的js文件，就看node_modules中有没有同名的文件夹
  5. 如果有同名文件夹，看里面是否有index.js,如果有就执行index.js
  6. 如果文件夹中没有index.js，就去package.json文件中的main选项中看一下入口文件是哪个
  7. 如果mian选项中没有指定的入口文件或找不到入口文件则报错
